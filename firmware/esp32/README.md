# Контроллер робота на ESP32

Прошивка подключает ESP32 к локальной Wi-Fi сети в режиме станции и принимает
команды backend-сервера по TCP на порту `3333`. Она управляет двумя двигателями,
двумя сервоприводами и хранит аварийную блокировку до явной команды `RESUME`.

## Настройка Wi-Fi

Скопируйте безопасный шаблон и укажите данные своей сети в локальной копии:

```powershell
Copy-Item include\wifi_credentials.example.h include\wifi_credentials.h
```

Файл `include/wifi_credentials.h` исключён из Git. Не помещайте SSID, пароль или
другие секреты в `src/main.cpp` и не коммитьте собранные файлы из `.pio/`.

Роутер должен выдать плате постоянный адрес или DHCP reservation. Этот адрес
задаётся для соответствующего агента в веб-интерфейсе.

## Сборка и загрузка

Установите [PlatformIO](https://platformio.org/), подключите ESP32 по USB и
выполните из этой папки:

```powershell
pio run
pio run --target upload
pio device monitor --baud 115200
```

## Протокол и защита

Поддерживаются команды движения, независимый PWM колёс для ручного режима,
границы поля, позиции, сервоприводы, штатный `HALT` и аварийный `STOP`.
Watchdog выключает двигатели, если сервер не обновлял команду движения в течение
одной секунды.

TCP-протокол не содержит аутентификации. Используйте контроллер только в
доверенной изолированной сети и обеспечьте физическую кнопку аварийного
отключения питания.
